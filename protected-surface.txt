# Every path here defines what "success" means: the gates, the thresholds,
# the policy configs, and the floor mechanism itself. Editing any of these
# requires the owner's review via the generated .github/CODEOWNERS.
#
# After changing this file, regenerate and commit CODEOWNERS:
#   python harness/structural-floor/gen_codeowners.py
# A sync test (and CI) fails the build if the two ever drift apart, so the
# success-definition surface cannot be quietly relocated.
